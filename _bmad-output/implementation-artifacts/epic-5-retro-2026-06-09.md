## What Went Well

1. **A pattern established once (5.1) paid dividends 3×.** The "pure module + injected accessor + module-level config resolver" shape (`viewstate.ts` `_preserveView`/`setPreserveView`/`getPreserveView`) was copied verbatim for `highlight_mode` (5.2), `search` (5.3), and `animate` (5.4). Decision D1 ("resolve config frontend-locally, no new wire surface") was the same decision four times with zero re-litigation — textbook scope discipline.
2. **The pure/DOM split made an otherwise-untestable feature testable.** All the logic — highlight sets, 4 highlight modes, search matching, invalid-regex sentinel, animation gate, reduced-motion fold — is pure and unit-tested (87 cases). Only the live SVG-emphasis DOM layer is manual. Maximum coverage achievable under the WASM-render constraint.
3. **`preserve_view` debt genuinely closed, not papered over.** A half-delivered v1 feature (config key since Story 2.1, never wired; README claim removed in `42738da`). Story 5.1 wired capture/restore into the render path for real and made the README claim true again.
4. **The hardest unknown resolved by reading source, not guessing.** Story 5.4's critical AC3 hazard (does `.on("end")` fire after a d3-graphviz transition, or does the render-lock get stuck?) was answered definitively by reading the pinned d3-graphviz 5.6.0 `render.js` — `"end"` fires last in both the transition and no-transition paths, so the render-lock releases exactly once. A *certain* answer despite no browser.
5. **Render-correctness invariants survived four stacked layers.** v-token guard + render-lock, last-good-render, error overlay, empty-buffer notice — every story re-verified them and hooked only into the single per-render success boundary (`renderDotWithFallback`), never the `onError` recovery path. No second concurrent `#app` mutation race was introduced.

## What Was Harder

1. **The browser render path still has no automated harness.** Now the project's #1 standing limitation (flagged in every Epic 5 story + project memory). ~200 lines of live DOM emphasis in `render.ts` are verified manually only. Story 5.4 shipped with four visual checks (tween-on/snap-off, fade highlight, latest-wins-under-load, no zoom jump) that no automated test covers.
2. **Config seams are frontend-local stubs, not real config.** `highlight_mode`, `search`, `interactive`/`animate` are all resolved with module-level frontend defaults — there is **no Lua config key** for any of them. A user cannot set `highlight_mode = "upstream"` from `setup()` today. The correct call per each story's AC (no new wire surface), but it means the FR-14 config story is deliberately deferred and currently invisible to users.
3. **An undeclared-dependency wart compounded into a pattern.** Story 5.1 imported `d3-zoom`'s `zoomTransform` undeclared (transitive-only); Story 5.4 did the same with `d3-transition`/`d3-ease`. Builds today (bundled as transitive deps of d3-graphviz), but fragile if d3-graphviz ever drops them.
4. **Story-file bookkeeping drifted.** Story 5.1's header still reads `Status: review` though it is committed (`13796a3`) and marked `done` in sprint-status. Cosmetic; worth a sweep.

## Cross-Story Patterns

- **Reuse-don't-reinvent was explicit and enforced.** Each story's Dev Notes named the exact seams to reuse (`renderDotWithFallback`, `extractModelFromApp`, `applyHighlightToDom`, `reapplyHighlightAfterRender`, the keydown predicate family) and forbade duplicating them. Search (5.3) added zero new extraction/emphasis machinery; animation (5.4) added zero new appliers.
- **The single shared emphasis regime held.** Click-highlight and search both flow through one `applyHighlightToDom` with one `ig-*` class set and a documented precedence (open search query owns the highlight; closing restores selection). No fighting parallel class regimes.
- **`Esc` was deliberately layered across three stories** (5.1 reset is `0`/`r`, not Esc; 5.2 clear-highlight; 5.3 close-search) using the INPUT/TEXTAREA-skip guard 5.2 left as a clean seam for 5.3 — no double-fire.

## Key Takeaways

1. **Picking the right dependency in v1 made an entire epic cheap.** d3-graphviz already shipped zoom, transitions, and the SVG; Epic 5 turned dormant capability on rather than building it. Architecture decisions compound.
2. **A reusable pattern beats four bespoke solutions.** The frontend-local config-resolver shape, defined once in 5.1, eliminated design cost in 5.2/5.3/5.4 entirely.
3. **Resolve unknowns with the strongest evidence available.** When no browser was available, reading the pinned dependency's source gave a *certain* answer to the render-lock lifecycle question — better than a deferred guess.
4. **Frontend-local stubs are a deliberate IOU, not a finished feature.** Four stories correctly avoided new wire surface — but the config is now invisible to users. Naming that as explicit debt (AI#2) keeps the IOU from being forgotten.
5. **The test gap is now load-bearing.** Four stories shipped real interactive UX whose live path is manual-only. Closing the browser-harness gap (AI#1) is the prerequisite to confidently iterating further.
