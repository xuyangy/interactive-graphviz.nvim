## What Went Well

1. **The headline feature works in the field.** Click a node → cursor jumps to the source line;
   rest the cursor → passive blue emphasis in the graph. The lead's day-to-day use (remote server,
   browser over SSH port-forward) confirms both directions with no reported friction.
2. **Written-down footguns did not become bugs.** Story prep explicitly named the traps before dev
   started — 6.3's `vim.NIL`-vs-`nil` clear frame ("#1 footgun": Lua's `json.encode` drops
   `nil`-valued keys, which would silently fail the server's `hasExactlyKeys` validation) and 6.4's
   `open_cmd` nil-default trap (invisible to `pairs(M.defaults)`). Both were implemented correctly
   on the first pass and pinned by dedicated tests.
3. **The spine-first sequencing paid off exactly as designed.** 6.1 shipped zero user-visible
   behavior but a fully contract-tested relay; 6.2/6.3 then activated it without ever touching the
   server again. RED-first testing in 6.1 (new spine tests failed against the pre-change server)
   proved the tests exercised the real relay.
4. **The adversarial review pipeline was productive on every story**: 5 patches (6.1), 5 patches +
   2 defers (6.2), two rounds with 4 patches + 5 defers (6.3), 5 patches (6.4). Highlights: matcher
   false-matches in comments/HTML labels, Unicode ID bytes (128–255), the `stop_all` fired-timer
   gap, watch-restart idempotency, and the click-during-debounce echo interleaving caught in 6.4.
5. **Epic 5 retro follow-through: 5/5 action items closed** (browser-DOM harness via happy-dom,
   real config keys + the URL-param contract, d3 sub-deps declared, bookkeeping sweep, CI busted
   glob). Most landed in the pre-epic debt paydown — the "pay down debt before v3" lead decision
   from the Epic 5 retro was executed as stated, and 6.1's contract test then locked down the very
   config plumbing that paydown created.
6. **Deferred-work triage worked as a system.** The 6.3 review deferred three hardening items with
   an explicit `→ Story 6.4` tag; 6.4 paid all three (per-buffer echo suppression, emission-time
   gate re-read, pcall symmetry) plus the vimdoc COMMANDS gap and warn-on-unknown-keys. Debt with a
   named landing story got paid; the ledger's ✅ annotations keep the audit trail.

## What Was Harder

1. **Pattern-mirroring copies bugs too.** `sync.lua`'s `stop_all` faithfully reproduced
   `render.lua`'s pre-existing fired-timer flaw (a fired debounce nils its timer entry, so
   steady-state watchers are skipped at teardown). Review caught sync's copy; **`render.stop_all`
   still has the original** (deferred, untagged — benign at its sole VimLeavePre call site).
   Lesson: when a story says "mirror the existing pattern verbatim," the pattern's known flaws ride
   along unless named.
2. **Two agents, one checkout (6.3).** Story 6.3 was implemented by two agents sharing the same
   working tree concurrently; overlaps were reconciled and the combined battery re-ran green, but
   6.4's story prep had to warn the next dev to verify a clean baseline. It worked out — via the
   test suite, not by design. Avoid repeating; if parallel agents are wanted, use worktree
   isolation.
3. **The reverse matcher accumulated known parity limitations** (attribute-key false-matches,
   concatenated quoted IDs, leading-dot numerals). All degrade gracefully by design (miss ≡ clear /
   informative notify) and are documented in deferred-work, but the per-line scanner is at the edge
   of what regex-ish scanning can do — a real token-level DOT scanner is the escalation path if
   these ever get reported in practice.
4. **The real-browser WASM render path remains unautomated** — the one standing limitation carried
   through every epic. Epic 6 widened what rides on it (sync round-trips) while verification stayed
   contract tests + happy-dom fixtures + manual checks.

## Key Takeaways

1. **Pre-paid architecture keeps compounding.** The v1 session token and the warm-but-dormant
   return channel made "activate bidirectional sync" a relay-case addition instead of a redesign —
   the same lesson as Epic 5's d3-graphviz choice, now proven on the protocol tier.
2. **Name the trap in the story and it won't ship as a bug.** Both epics' costliest potential
   defects (`vim.NIL`, `open_cmd` nil-default) were neutralized by story-prep that called them out
   with mechanism, not just warning.
3. **Tagged deferral beats untracked deferral.** Every `→ Story 6.4` item got paid; the untagged
   `render.stop_all` twin of a fixed bug is still open. Debt with a named landing story is debt
   that gets repaid.
4. **"Mirror the pattern" needs a flaw check.** Verbatim pattern reuse propagated a known teardown
   bug into new code; reviews caught the copy but not (yet) the original.
5. **Don't share a checkout between concurrent agents.** 6.3's overlap was absorbed by the test
   suite, not prevented by process; use worktrees next time.
