CONFIGURATION                                 *interactive-graphviz-config*

Call `setup()` with any subset of the keys below (defaults shown). Invalid
values are rejected with a warning and fall back to the default rather than
failing setup. >lua

    require("interactive-graphviz").setup({
      engine = "dot",            -- default engine; must be in `engines`
      engines = { "dot", "neato" }, -- selectable engines
      debounce_ms = 200,         -- live-reload debounce (ms), > 0
      port = 0,                  -- listen port; 0 = ephemeral (OS-assigned)
      expose_to_lan = false,     -- false = 127.0.0.1; true = 0.0.0.0
      open_cmd = nil,            -- nil = vim.ui.open; or a command string
      preserve_view = true,      -- keep zoom/pan across re-renders
      highlight_mode = "bidirectional", -- single | upstream | downstream
                                        -- | bidirectional
      animate = true,            -- animate re-renders + highlight changes
      search = {                 -- initial state of the live-search (`/`) box
        scope = "both",          -- both | nodes | edges
        case_sensitive = false,  -- start with the Aa toggle on
        regex = false,           -- start with the .* toggle on
      },
      sync = {                   -- editor<->graph sync
        jump_on_click = true,    -- click node -> cursor jumps to source line
        highlight_on_cursor = true, -- cursor on node/edge line -> outline it
        cursor_debounce_ms = 150,   -- cursor-sync debounce (ms), > 0
      },
      heartbeat_ms = 2000,       -- supervision heartbeat interval (ms), > 0
      log_level = "warn",        -- off | error | warn | info | debug
    })
<
`bind` is not a user-settable key — the bind address is controlled
exclusively by `expose_to_lan`. Unknown keys (including typos in
`search`/`sync` subfields) warn naming the offending key and are ignored.

INTERACTIVITY KEYS                     *interactive-graphviz-interactivity*

                                          *interactive-graphviz-view-controls*
Browser view controls ~
    In the preview, the view is driven by mouse gestures — scroll wheel to
    zoom, double-click to zoom in, Shift+double-click to zoom out, click+drag
    to pan, `0` or `r` to reset the view to fit the viewport, `Shift+F` to
    fit the whole graph into the current window (recomputed live, so it is
    the one to reach for after resizing the browser window), and `p` to
    toggle pan-scroll mode — the wheel then pans (scroll up/down,
    Shift+scroll left/right) instead of zooming; double-click still zooms.
    A toolbar at the top-right offers the same as buttons: a home icon to
    reset-to-fit, a fit icon (corner brackets) to fit the graph to the
    window, +/- magnifier icons to zoom, a four-way-arrows icon that
    toggles pan-scroll mode, plus a download icon that saves the
    graph as `graph.svg` — exported as currently rendered (zoom/pan
    included; reset the view first for the full graph), with any transient
    click/cursor emphasis stripped. A final icon saves a self-contained
    interactive `graph.html` (~1 MB): it opens offline and keeps zoom/pan,
    click-highlight, `/` search and the SVG export working, re-rendering
    the graph fresh when opened. Neovim-linked features (jump-on-click,
    cursor echo, live reload) are inert in the exported file, and the file
    is safe to share: it carries the interactivity config but never the
    preview session's auth token. See the README for the full gesture
    table.

                                          *interactive-graphviz-preserve_view*
preserve_view ~
    Boolean, default `true`. Keep the current zoom/pan across live-reload
    re-renders. Set `false` to reset to fit-to-viewport on every reload.

                                         *interactive-graphviz-highlight_mode*
highlight_mode ~
    String enum, default `"bidirectional"`. Which neighbors light up when a
    node is clicked in the preview:

        `single`         just the clicked node (no neighbors)
        `upstream`       predecessors + the connecting edges
        `downstream`     successors + the connecting edges
        `bidirectional`  both directions (default)

                                                *interactive-graphviz-animate*
animate ~
    Boolean, default `true`. Animate graph re-renders and highlight changes.
    Cursor emphasis also continuously blooms outward and contracts inward
    while its target remains active.
    Set `false` for instant updates. The OS-level
    `prefers-reduced-motion: reduce` preference always disables animation
    regardless of this key. With motion disabled, cursor emphasis remains a
    strong, steady cyan outline and glow.

                                                 *interactive-graphviz-search*
search ~
    Table; the initial state of the live-search box (opened with `/` in the
    preview). Partial tables are fine — unset subfields keep their defaults:

        `scope`           `"both"` | `"nodes"` | `"edges"` (default `"both"`)
        `case_sensitive`  boolean (default `false`)
        `regex`           boolean (default `false`)

                                                   *interactive-graphviz-sync*
sync ~
    Table; editor<->graph sync (both directions). Partial tables are fine —
    unset subfields keep their defaults:

        `jump_on_click`       boolean (default `true`). Clicking a node in
                              the preview moves the Neovim cursor to the
                              node's first source line. OS window focus stays
                              in the browser — the plugin never raises the
                              Neovim window.
        `highlight_on_cursor` boolean (default `true`). Resting the cursor on
                              a node's line gives that node a conspicuous cyan
                              glow bloom in the preview; on an edge line
                              (`a -> b`) the edge AND both endpoint nodes glow,
                              whatever the column (chains light the segment
                              under the cursor). A line with no node or edge
                              clears it.
                              Click/search stroke precedence stays
                              authoritative, and unrelated elements are never
                              dimmed. With animation disabled or reduced
                              motion requested, the same targets keep a strong
                              steady glow.
                              If the target is off-screen the view pans to
                              center it (same zoom level); an on-screen
                              target never moves the view. Ports and
                              subgraph endpoints fall back to single-node
                              outlining.
        `cursor_debounce_ms`  positive integer (default `150`). Debounce for
                              the cursor->graph direction, in milliseconds.

                                     *interactive-graphviz-config-apply-rule*
When config applies ~
    The interactivity keys (`preserve_view`, `highlight_mode`, `animate`,
    `search`) are carried to the browser in the preview URL, so they apply
    when a preview OPENS. An already-open preview does not pick up a changed
    value live — and note that reloading the tab re-applies the OLD config
    baked into that tab's URL, not your latest `setup()`. To pick up a
    change, re-open the preview (`:GraphvizPreviewStop`, then
    `:GraphvizPreview`).

    The sync keys split across that line: `sync.jump_on_click` rides the
    preview URL like the interactivity keys (applies at preview open; a tab
    reload re-applies the value baked into the URL), while
    `sync.highlight_on_cursor` and `sync.cursor_debounce_ms` are read on the
    Neovim side — a `setup()` change applies from the next cursor movement,
    no re-open needed. One nuance when turning `highlight_on_cursor` off
    mid-session: new outlines stop immediately, but an outline already on
    screen stays until the next preview open or stop (emphasis frames are
    transient and last-wins; nothing repaints on a config change alone).
    |:GraphvizCursorHighlightToggle| does not have this nuance — it clears the
    on-screen outline as part of turning the feature off.

    Both sync gates can also be flipped at runtime without a `setup()` call via
    |:GraphvizCursorHighlightToggle| and |:GraphvizJumpOnClickToggle|.

==============================================================================
COMMANDS                                       *interactive-graphviz-commands*

                                                            *:GraphvizPreview*
:GraphvizPreview
    Start the preview for the current DOT buffer and open it in the browser.

                                                        *:GraphvizPreviewStop*
:GraphvizPreviewStop
    Stop the current buffer's preview. Idempotent; shuts the server down when
    it was the last session.

                                                      *:GraphvizPreviewToggle*
:GraphvizPreviewToggle
    Start the preview if stopped, stop it if running.

                                                             *:GraphvizEngine*
:GraphvizEngine [engine]
    With no argument, report the current and available layout engines. With
    an argument (e.g. `:GraphvizEngine neato`), switch the layout engine and
    re-render.

                                          *:GraphvizCursorHighlightToggle*
:GraphvizCursorHighlightToggle
    Toggle `sync.highlight_on_cursor` on/off at runtime (buffer->graph: moving
    the cursor onto a node/edge line outlines it in the preview). Applies to
    every open preview at once; turning it off also clears any outline already
    on screen. A runtime override like |:GraphvizEngine| — a later `setup()`
    resets it.

                                              *:GraphvizJumpOnClickToggle*
:GraphvizJumpOnClickToggle
    Toggle `sync.jump_on_click` on/off at runtime (graph->buffer: clicking a
    node in the preview moves the Neovim cursor to its source line). Pushed to
    every open preview live via a config_update — no re-open needed. A runtime
    override like |:GraphvizEngine| — a later `setup()` resets it.

                                                                *:GraphvizUrl*
:GraphvizUrl
    Print the current buffer's full preview URL into |:messages| history.
    Useful when the startup notification was truncated or swallowed by a
    notification UI — the echoed URL stays retrievable via |:messages|.
    Requires an active preview. Note the URL contains the session's auth
    token (loopback-scoped, dies with the server).

                                               *interactive-graphviz-health*
:checkhealth interactive-graphviz
    Diagnostics: Neovim version, checksums presence, the mapped prebuilt
    binary's presence and checksum, Bun availability (for the source-build
    fallback), and localhost port-bind capability. See |health|.
